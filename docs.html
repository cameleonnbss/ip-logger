📦 Installation

One command on every platform. cloudflared is auto-installed on first run if missing.

Windows
install.bat

Installs cloudflared via winget (fallback: direct download). Requires Python 3.8+.

Linux / macOS
bash install.sh
python3 ip_logger.py

Auto-detects arch (amd64 / arm64 / arm). Works on Debian, Ubuntu, Arch, Fedora, macOS.

Termux (Android)
pkg update && pkg install python
bash install.sh
python ip_logger.py

Downloads cloudflared ARM binary directly. No root required.

ℹ️ If cloudflared is already installed it won't be reinstalled. The script checks before downloading.

🎭 Lure pages

10 fake pages that run the beacon script before redirecting. Each collects all available data while looking legitimate.

KeyDescriptionRedirect delay

💡 The beacon uses a synchronous XHR to guarantee data is sent before the redirect fires — stats are never lost even on slow connections.
↩️ Modes
🎭
Lure mode default
Shows a fake page (captcha, Google Drive, etc.) while collecting all data in the background. Best for social engineering.
↩️
Redirect mode
Instant 302 redirect to the target URL. Still runs the beacon in a micro-page before redirecting. Less suspicious for short links.
🌍 GeoIP

Three sources in cascade. If one fails or rate-limits, the next is tried automatically.

SourceFieldsKey detail
1st ip-api.com country, region, city, district, ZIP, ISP, org, ASN, lat/lon, mobile, proxy, hosting Proxy/VPN/hosting flag
2nd freeipapi.com country, region, city, ZIP, lat/lon No key required
3rd ipinfo.io country, region, city, postal, org, lat/lon Fallback
ℹ️ GeoIP runs in a background thread — it never delays the response to the target. Data is updated in the log file once enrichment completes.
🕵️ Fingerprinting

The beacon script runs silently in the browser and sends a rich fingerprint before any redirect.

🖥️
Hardware
Screen resolution, available size, DPR, RAM (deviceMemory), CPU core count, battery level & charging state.
🌐
Browser
Browser name/version, vendor, platform, plugin list (up to 6), User-Agent, language & locale chain.
🎨
Canvas & WebGL
Canvas fingerprint (last 32 chars of data URL), WebGL renderer + vendor — unique per GPU/driver combination.
📶
Network
Effective connection type (4G/WiFi/3G...), downlink speed, touch capability, touch point count.
🗺️
GPS
Precise coordinates via browser Geolocation API (lat, lon, ± accuracy in meters). Only collected when already granted — no popup on most mobile browsers.
🔧
Misc
Timezone, cookies enabled, Do Not Track flag, timezone from Intl API.
🌐 Cloudflare Tunnel

Quick tunnel — no account needed. Gets you a public *.trycloudflare.com URL in seconds.

⚡
Auto-install
If cloudflared is not found, it's installed automatically via winget (Windows), Homebrew (macOS), or direct binary download (Linux/Termux).
🔄
Auto-start
Tunnel starts with the server. The public URL is displayed as soon as it's detected from cloudflared's output (stdout + stderr).
🎛️
Toggle in menu
Option 8 in the menu lets you cut and restart the tunnel at any time without restarting the server or losing your logs.
⚠️ Quick tunnel URLs are ephemeral — they change on every restart. Reconnecting gives a new URL.
📊 Dashboard

Accessible at http://localhost:PORT/logs — dark UI with stat cards, sortable table, bar charts, and a per-capture detail modal.

📋
Captures tab
All visits in reverse order. Click any row to open a full detail modal with 30+ fields. Live search bar filters across all columns.
📈
Stats tab
Bar charts: top countries, browsers, operating systems, devices. Percentages calculated in real-time from log data.
⚙️
Config tab
Shows active tracking URLs, pixel URL, current lure, redirect target and log file path. One-click copy buttons.
🔒 Dashboard auth

Password protection uses HTTP Basic Auth — enforced server-side. The password never appears in the HTML source or in URLs.

ScenarioBehavior
No password setDashboard is public — accessible without auth
Password setBrowser shows a native login popup (HTTP 401 + WWW-Authenticate header)
Wrong password401 returned, browser prompts again

Set via option 6 in the CLI menu. Saved to iplogger_config.json.

💾 Config file

Settings are auto-saved to iplogger_config.json after every change in the menu. Loaded automatically on next start.

{
  "port": 8080,
  "mode": "lure",
  "lure": "captcha",
  "redirect": "https://www.youtube.com",
  "secret": "yourpassword",
  "track_token": "abc123xyz",
  "lang": "en"
}
ℹ️ iplogger_config.json and ip_log.json are in .gitignore — they won't be pushed to the repo.
